Klassenfragen

1. Object

Die Klasse Object ist die Superklasse aller Klassen in Java. Sie stellt grundlegende Methoden bereit, die alle Objekte haben.

  • Wichtige Methoden:
    • boolean equals(Object obj)
    • String toString()
    • int hashCode()
    • Object clone()
  • Codebeispiel:
    public class MyObjectClass {
    private int id;

    public MyClass(int id) {
        this.id = id;
    }

    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;
        if (obj == null || getClass() != obj.getClass()) return false;
        MyClass myClass = (MyClass) obj;
        return id == myClass.id;
    }

    @Override
    public int hashCode() {
        return Objects.hash(id);
    }

    @Override
    public String toString() {
        return "MyClass{id=" + id + "}";
    }
}

Erweiterte Methoden und Konstrukte in Java

Zusätzlich zu den grundlegenden Methoden, die in der vorherigen Lernübersicht beschrieben wurden, gibt es erweiterte Methoden und Konstrukte wie forEach, removeIf und andere, die besonders nützlich in der Arbeit mit Kollektionen sind.

forEach

forEach ist eine Methode, die in vielen Klassen des java.util-Pakets verfügbar ist, insbesondere in den Klassen, die das Iterable-Interface implementieren (z.B. ArrayList, HashSet). Diese Methode ermöglicht es, eine bestimmte Operation auf jedes Element der Sammlung auszuführen.

  • Codebeispiel:
    javaCode kopieren
import java.util.ArrayList;
import java.util.List;

public class Main {
    public static void main(String[] args) {
        List<String> list = new ArrayList<>();
        list.add("A");
        list.add("B");
        list.add("C");

        // Verwendung von forEach mit einer Lambda-Ausdruck
        list.forEach(element -> System.out.println(element));
    }
}

removeIf

removeIf ist eine Methode, die es ermöglicht, Elemente aus einer Sammlung zu entfernen, die einer bestimmten Bedingung entsprechen. Diese Methode verwendet eine Predicate-Schnittstelle, die eine Bedingung als Lambda-Ausdruck oder Methodenreferenz akzeptiert.

  • Codebeispiel:
    import java.util.ArrayList;
import java.util.List;

public class Main {
    public static void main(String[] args) {
        List<String> list = new ArrayList<>();
        list.add("Apple");
        list.add("Banana");
        list.add("Cherry");

        // Entfernt alle Elemente, die mit 'B' beginnen
        list.removeIf(element -> element.startsWith("B"));

        list.forEach(element -> System.out.println(element));
        // Output: Apple, Cherry
    }
}

Weitere wichtige Methoden

stream

Streams bieten eine Möglichkeit, auf deklarative Weise mit Sammlungen zu arbeiten, indem sie Methoden wie filter, map, collect und andere verwenden. Streams sind Teil der Java Stream API, die mit Java 8 eingeführt wurde.

  • Codebeispiel:
    import java.util.ArrayList;
import java.util.List;
import java.util.stream.Collectors;

public class Main {
    public static void main(String[] args) {
        List<String> list = new ArrayList<>();
        list.add("Apple");
        list.add("Banana");
        list.add("Cherry");

        // Filtert die Liste und sammelt nur die Elemente, die mit 'A' beginnen
        List<String> filteredList = list.stream()
                                        .filter(element -> element.startsWith("A"))
                                        .collect(Collectors.toList());

        filteredList.forEach(element -> System.out.println(element));
        // Output: Apple
    }
}

map

Die Methode map wird verwendet, um Elemente eines Streams in eine andere Form zu transformieren. Sie nimmt eine Funktion als Parameter, die auf jedes Element angewendet wird.

  • Codebeispiel:
    import java.util.ArrayList;
import java.util.List;
import java.util.stream.Collectors;

public class Main {
    public static void main(String[] args) {
        List<String> list = new ArrayList<>();
        list.add("apple");
        list.add("banana");
        list.add("cherry");

        // Konvertiert jedes Element in Großbuchstaben
        List<String> upperCaseList = list.stream()
                                         .map(String::toUpperCase)
                                         .collect(Collectors.toList());

        upperCaseList.forEach(element -> System.out.println(element));
        // Output: APPLE, BANANA, CHERRY
    }
}

filter

Die Methode filter wird verwendet, um Elemente eines Streams basierend auf einer Bedingung zu filtern. Sie nimmt ein Predicate als Parameter.

  • Codebeispiel:
    import java.util.ArrayList;
import java.util.List;
import java.util.stream.Collectors;

public class Main {
    public static void main(String[] args) {
        List<String> list = new ArrayList<>();
        list.add("apple");
        list.add("banana");
        list.add("cherry");

        // Filtert die Liste und behält nur die Elemente, die mehr als 5 Zeichen haben
        List<String> longWords = list.stream()
                                     .filter(element -> element.length() > 5)
                                     .collect(Collectors.toList());

        longWords.forEach(element -> System.out.println(element));
        // Output: banana, cherry
    }
}

Zusammenfassung

Diese erweiterten Methoden und Konstrukte bieten eine leistungsfähige und ausdrucksstarke Möglichkeit, mit Kollektionen in Java zu arbeiten. forEach und removeIf sind besonders nützlich für das Arbeiten mit Sammlungen, während Streams und ihre Methoden wie map, filter und collect eine deklarative Programmierung ermöglichen. Diese Konzepte sind entscheidend für das Verständnis moderner Java-Programmierung und sollten gründlich verstanden und geübt werden.

Vektoren, Matrizen und Binärumwandlung

1. Vektoren

In Java werden Vektoren häufig durch die Klasse ArrayList oder Vector dargestellt. Beide Klassen implementieren die List-Schnittstelle, wobei Vector synchronisiert ist und daher thread-sicher.

  • Wichtige Methoden von ArrayList:
    • boolean add(E e)
    • E get(int index)
    • E remove(int index)
    • int size()
  • Codebeispiel mit ArrayList:
    javaCode kopieren
import java.util.ArrayList;

public class VectorExample {
    public static void main(String[] args) {
        ArrayList<Integer> vector = new ArrayList<>();
        vector.add(1);
        vector.add(2);
        vector.add(3);

        System.out.println("Element at index 1: " + vector.get(1)); // Output: 2

        vector.remove(1);

        System.out.println("Vector size after removal: " + vector.size()); // Output: 2

        for (int element : vector) {
            System.out.println(element); // Output: 1, 3
        }
    }
}

2. Matrizen

Matrizen können in Java durch zweidimensionale Arrays dargestellt werden. Hier wird die Speicherung und der Zugriff auf Elemente einer Matrix demonstriert.

  • Speicherung und Zugriff auf Matrizen:
    javaCode kopieren
public class MatrixExample {
    public static void main(String[] args) {
        // Initialisierung einer 3x3 Matrix
        int[][] matrix = {
            {1, 2, 3},
            {4, 5, 6},
            {7, 8, 9}
        };

        // Zugriff auf ein Element der Matrix
        System.out.println("Element at (1, 1): " + matrix[1][1]); // Output: 5

        // Ändern eines Elements der Matrix
        matrix[1][1] = 10;

        // Ausgabe der gesamten Matrix
        for (int i = 0; i < matrix.length; i++) {
            for (int j = 0; j < matrix[i].length; j++) {
                System.out.print(matrix[i][j] + " ");
            }
            System.out.println();
        }
        // Output:
        // 1 2 3
        // 4 10 6
        // 7 8 9
    }
}

3. Binärumwandlung

Die Binärumwandlung in Java kann mit Hilfe der Klasse Integer durchgeführt werden, die Methoden zur Konvertierung von Dezimalzahlen in Binärzahlen und umgekehrt bietet.

  • Wichtige Methoden der Klasse Integer:
    • static String toBinaryString(int i) (gibt auch toOctalString …)
    • static int parseInt(String s, int radix)
  • Codebeispiel für Binärumwandlung:
    javaCode kopieren
public class BinaryConversionExample {
    public static void main(String[] args) {
        int number = 42;
        String binaryString = Integer.toBinaryString(number);

1. Exception-Handling

Exception-Handling in Java ermöglicht es, Laufzeitfehler zu erkennen und darauf zu reagieren, ohne dass das Programm abrupt beendet wird. Es gibt zwei Haupttypen von Exceptions: Checked Exceptions und Unchecked Exceptions.

  • Wichtige Konzepte:
    • trycatchBlöcke
    • finallyBlock
    • Eigene Ausnahme erstellen (extends Exception oder extends RuntimeException)
  • Codebeispiel:
    public class ExceptionHandlingExample {
    public static void main(String[] args) {
        try {
            int result = divide(10, 0);
            System.out.println("Result: " + result);
        } catch (ArithmeticException e) {
            System.err.println("Error: Division by zero!");
        } finally {
            System.out.println("This block is always executed.");
        }
    }

    public static int divide(int a, int b) throws ArithmeticException {
        return a / b;
    }
}

2. Lambda-Ausdrücke

Lambda-Ausdrücke in Java sind eine Möglichkeit, anonyme Methoden zu schreiben. Sie ermöglichen eine klarere und prägnantere Syntax, insbesondere beim Umgang mit funktionalen Schnittstellen.

  • Wichtige Konzepte:
    • Syntax: (parameters) -> expression oder (parameters) -> { statements }
    • Funktionale Schnittstellen (z.B. Runnable, Callable, Comparator)
  • Codebeispiel:
    import java.util.Arrays;
import java.util.List;

public class LambdaExample {
    public static void main(String[] args) {
        List<String> names = Arrays.asList("Alice", "Bob", "Charlie");

        // Verwendung von Lambda-Ausdrücken
        names.forEach(name -> System.out.println(name));

        // Mit funktionaler Schnittstelle
        Thread thread = new Thread(() -> System.out.println("Hello from a thread!"));
        thread.start();
    }
}

3. Streams & deren Operationen

Die Stream-API bietet eine moderne Methode zur Verarbeitung von Sammlungen und Arrays. Streams unterstützen eine Vielzahl von Operationen wie Filterung, Mapping und Reduktion.

  • Wichtige Operationen:
    • filter(Predicate<? super T> predicate)
    • map(Function<? super T, ? extends R> mapper)
    • collect(Collector<? super T, A, R> collector)
  • Codebeispiel:
    javaCode kopieren
import java.util.Arrays;
import java.util.List;
import java.util.stream.Collectors;

public class StreamExample {
    public static void main(String[] args) {
        List<String> names = Arrays.asList("Alice", "Bob", "Charlie", "David");

        // Filtern und Sammeln
        List<String> filteredNames = names.stream()
                                          .filter(name -> name.startsWith("A"))
                                          .collect(Collectors.toList());

        filteredNames.forEach(System.out::println); // Output: Alice

        // Mapping und Sammeln
        List<String> upperCaseNames = names.stream()
                                           .map(String::toUpperCase)
                                           .collect(Collectors.toList());

        upperCaseNames.forEach(System.out::println); // Output: ALICE, BOB, CHARLIE, DAVID
    }
}

4. Eigene Datenklassen erstellen

Eigene Datenklassen enthalten Felder, Konstruktoren, Getter und Setter, sowie überschriebenen Methoden wie toString, equals und hashCode.

  • Wichtige Konzepte:
    • Felder und Konstruktoren
    • Getter und Setter
    • Überschreiben von toString(), equals(Object obj) und hashCode()
  • Codebeispiel:
    javaCode kopieren
public class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    @Override
    public String toString() {
        return "Person{name='" + name + "', age=" + age + "}";
    }

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        Person person = (Person) o;
        return age == person.age && Objects.equals(name, person.name);
    }

    @Override
    public int hashCode() {
        return Objects.hash(name, age);
    }
}

public class Main {
    public static void main(String[] args) {
        Person person1 = new Person("Alice", 30);
        Person person2 = new Person("Bob", 25);

        System.out.println(person1);
        System.out.println(person2);

        // Vergleich von Personen
        System.out.println(person1.equals(person2)); // Output: false
    }
}

Ereignisbehandlung

  • ActionListener: Reagiert auf Aktionen wie das Klicken auf einen Button.
  • WindowListener: Reagiert auf Fensterereignisse wie das Schließen eines Fensters.

Beispiel: Schließen des Fensters mit Bestätigung:

import javax.swing.*;
import java.awt.*;
import java.awt.event.*;

public class ConfirmCloseWindow extends JFrame {
    public ConfirmCloseWindow() {
        setTitle("Confirm Close Window");
        setLayout(new GridLayout(4, 5));

        for (int i = 1; i <= 20; i++) {
            add(new JButton(String.valueOf(i)));
        }

        addWindowListener(new WindowAdapter() {
            public void windowClosing(WindowEvent e) {
                int confirmed = JOptionPane.showConfirmDialog(null,
                  "Are you sure you want to exit?", "Exit Confirmation",
                  JOptionPane.YES_NO_OPTION);

                if (confirmed == JOptionPane.YES_OPTION) {
                    dispose();
                }
            }
        });

        setSize(400, 300);
        setDefaultCloseOperation(JFrame.DO_NOTHING_ON_CLOSE);
        setVisible(true);
    }

    public static void main(String[] args) {
        new ConfirmCloseWindow();
    }
}

Benutzerdefinierte Zeichenfläche

  • JPanel: Kann überschrieben werden, um benutzerdefinierte Grafiken zu zeichnen.

Beispiel: Zeichenfläche mit Mausereignissen:

import javax.swing.*;
import java.awt.*;
import java.awt.event.*;

public class CustomPanel extends JPanel {
    private boolean insideSquare = false;

    public CustomPanel() {
        setPreferredSize(new Dimension(400, 400));
        addMouseMotionListener(new MouseAdapter() {
            public void mouseMoved(MouseEvent e) {
                int x = e.getX();
                int y = e.getY();
                insideSquare = x >= 50 && x <= 100 && y >= 100 && y <= 150;
                repaint();
            }
        });
    }

    protected void paintComponent(Graphics g) {
        super.paintComponent(g);
        g.setColor(insideSquare ? Color.RED : Color.GREEN);
        g.fillRect(50, 100, 50, 50);
    }

    public static void main(String[] args) {
        JFrame frame = new JFrame("Custom Panel");
        frame.add(new CustomPanel());
        frame.pack();
        frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        frame.setVisible(true);
    }
}

1. Schichtenarchitektur

Die Schichtenarchitektur trennt die Anwendung in mehrere Schichten, die jeweils eine spezifische Verantwortung haben. Die häufigsten Schichten sind:

  • Präsentationsschicht: Verarbeitet die Benutzeroberfläche und Benutzerinteraktionen.
  • Geschäftslogikschicht: Enthält die Geschäftslogik und Anwendungsregeln.
  • Datenzugriffsschicht: Kümmert sich um die Datenbankoperationen.
  • Codebeispiel:
    // Präsentationsschicht
public class Main {
    public static void main(String[] args) {
        UserService userService = new UserService(new UserRepository());
        User user = userService.getUserById(1);
        System.out.println(user.getName());
    }
}

// Geschäftslogikschicht
public class UserService {

        this.userRepository = userRepository;

    public User getUserById(int id) {
        return userRepository.findById(id);
    }
}

// Datenzugriffsschicht
public class UserRepository {

2. SOLID-Prinzipien

Die SOLID-Prinzipien sind Designprinzipien, die helfen, wartbaren und erweiterbaren Code zu schreiben.

  • Single Responsibility Principle (SRP): Eine Klasse sollte nur eine Verantwortlichkeit haben.
  • Open/Closed Principle (OCP): Klassen sollten für Erweiterungen offen, aber für Änderungen geschlossen sein.
  • Liskov Substitution Principle (LSP): Objekte einer Basisklasse sollten durch Objekte einer abgeleiteten Klasse ersetzbar sein.
  • Interface Segregation Principle (ISP): Viele spezifische Schnittstellen sind besser als eine allgemeine.
  • Dependency Inversion Principle (DIP): Abstraktionen sollten nicht von konkreten Implementierungen abhängen.
  • Beispiel für SRP:
    public class User {
    private String name;

    public User(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }
}

public class UserPrinter {
    public void printUserName(User user) {
        System.out.println(user.getName());
    }
}

3. Containerklassen

Containerklassen sind Klassen, die andere Objekte enthalten und verwalten, z.B. ArrayList, HashMap, HashSet. (kann auch ohne main() Funktion sein)

    import java.util.ArrayList;
import java.util.HashMap;

        list.add("Alice");
        list.add("Bob");

    }
}

SQL-Befehle und deren Verwendung

1. Datenbank und Tabellenverwaltung

CREATE DATABASE

  • Erstellt eine neue Datenbank.
CREATE DATABASE datenbank_name;

CREATE TABLE

  • Erstellt eine neue Tabelle.

DROP DATABASE

  • Löscht eine bestehende Datenbank.
DROP DATABASE datenbank_name;

DROP TABLE

  • Löscht eine bestehende Tabelle.
DROP TABLE tabelle_name;

ALTER TABLE

  • Ändert die Struktur einer bestehenden Tabelle.
ALTER TABLE tabelle_name
ADD spalte_name datentyp;

2. Datenmanipulation

INSERT INTO

  • Fügt neue Daten in eine Tabelle ein.
INSERT INTO Vorlesung
VALUES (NULL, ’Informatik 2’, ’2022’, ’SoSe’, 13);

UPDATE

  • Aktualisiert vorhandene Daten in einer Tabelle.
UPDATE Vorlesung
SET DozentID = 5
WHERE ID = 137;

DELETE

  • Löscht Daten aus einer Tabelle.
DELETE FROM Vorlesung
WHERE Semester = ’SoSe’;

3. Datenabfrage

SELECT

  • Wählt Daten aus einer Tabelle aus.
SELECT FROM Vorlesung
WHERE DozentID IS NULL;

JOIN

  • Kombiniert Zeilen aus zwei oder mehr Tabellen basierend auf einer verwandten Spalte zwischen ihnen.
SELECT tabelle1.spalte1, tabelle2.spalte2, ...
FROM tabelle1
JOIN tabelle2
ON tabelle1.gemeinsame_spalte = tabelle2.gemeinsame_spalte;

WHERE

  • Filtert Datensätze basierend auf einer Bedingung.
SELECT spalte1, spalte2, ...
FROM tabelle_name
WHERE bedingung;

// DISTINCT -> ohne duplikate
SELECT DISTINCT Name, Semester FROM Vorlesung;

GROUP BY

  • Gruppiert Zeilen, die den gleichen Wert in den angegebenen Spalten haben.
SELECT spalte1, COUNT(*)
FROM tabelle_name
GROUP BY spalte1;

ORDER BY

  • Sortiert die Ergebniszeilen in auf- oder absteigender Reihenfolge.
SELECT spalte1, spalte2, ...
FROM tabelle_name
ORDER BY spalte1 ASC|DESC;

HAVING

  • Filtert Gruppen von Zeilen.
SELECT spalte1, COUNT(*)
FROM tabelle_name
GROUP BY spalte1
HAVING COUNT(*) > wert;

4. Datenintegrität

PRIMARY KEY

  • Definiert einen Primärschlüssel. Er stellt sicher, dass jede Zeile in einer Tabelle eindeutig identifizierbar ist.
CREATE TABLE tabelle_name (
    spalte1 datentyp PRIMARY KEY,
    ...
);

FOREIGN KEY

  • Definiert einen Fremdschlüssel. Er stellt eine Beziehung zwischen zwei Tabellen her und sorgt für referenzielle Integrität.
CREATE TABLE tabelle_name (
    spalte1 datentyp,
    spalte2 datentyp,
    FOREIGN KEY (spalte1) REFERENCES andere_tabelle(spalte1)
);

UNIQUE

  • Erzwingt die Einzigartigkeit von Werten in einer Spalte, sodass keine doppelten Werte gespeichert werden können.
CREATE TABLE tabelle_name (
    spalte1 datentyp UNIQUE,
    ...
);

CHECK

  • Erzwingt eine Bedingung für Werte in einer Spalte, um sicherzustellen, dass nur gültige Daten eingegeben werden.
CREATE TABLE tabelle_name (
    spalte1 datentyp,
    spalte2 datentyp CHECK (spalte2 > 0),
    ...
);

1. Threads

Ein Thread ist der kleinste Ausführungseinheit innerhalb eines Prozesses. Java bietet Unterstützung für Multithreading, was die gleichzeitige Ausführung mehrerer Threads innerhalb eines Programms ermöglicht.

  • Thread erstellen und starten:
    • Mit der ThreadKlasse.
    • Mit der RunnableSchnittstelle.
  • Codebeispiel:
    public class MyThread extends Thread {
    @Override
    public void run() {
        for (int i = 0; i < 5; i++) {
            System.out.println("Thread: " + i);
            try {
                Thread.sleep(1000); // 1 Sekunde warten
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }

    public static void main(String[] args) {
        MyThread thread = new MyThread();
        thread.start();
    }
}

2. Runnable

Runnable ist eine funktionale Schnittstelle, die eine einzige Methode run() enthält. Sie wird verwendet, um den Code zu definieren, der von einem Thread ausgeführt werden soll, ohne die Thread-Klasse zu erweitern.

  • Codebeispiel:
    public class MyRunnable implements Runnable {
    @Override
    public void run() {
        for (int i = 0; i < 5; i++) {
            System.out.println("Runnable: " + i);
            try {
                Thread.sleep(1000); // 1 Sekunde warten
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }

    public static void main(String[] args) {
        Thread thread = new Thread(new MyRunnable());
        thread.start();
    }
}

3. Countdown Example

Ein Countdown-Timer ist ein gutes Beispiel für die Anwendung von Threads. Hier wird ein Countdown von 10 Sekunden implementiert, der in jeder Sekunde aktualisiert wird.

  • Codebeispiel:
    public class CountdownTimer implements Runnable {
    private int seconds;

    public CountdownTimer(int seconds) {
        this.seconds = seconds;
    }

    @Override
    public void run() {
        for (int i = seconds; i >= 0; i--) {
            System.out.println("Countdown: " + i);
            try {
                Thread.sleep(1000); // 1 Sekunde warten
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        System.out.println("Countdown completed!");
    }

    public static void main(String[] args) {
        int countdownSeconds = 10;
        Thread countdownThread = new Thread(new CountdownTimer(countdownSeconds));
        countdownThread.start();
    }
}
